# Admin Scaffold + Sanity Seam Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up `apps/admin` (TanStack Start) with the Sanity seam — the server-only layer
(`server/sanity/*`, `server/functions/*`) plus shared zod schemas/types/formatters — so later
plans can build UI features against a working, tested data layer.

**Architecture:** TanStack Start app in the pnpm workspace. All Sanity I/O happens inside
`createServerFn` handlers that call `server/sanity/*` modules; the browser never imports
`@sanity/client` or sees the token. `shared/schemas.ts` (zod) is the single source of truth for
document shapes, imported by both server validators and (later) client forms.

**Tech Stack:** TanStack Start 1.168.27, TanStack Router 1.170.17, TanStack Query 5.101.2,
`@tanstack/router-plugin` 1.168.19, `@sanity/client` ^7.23.0 (match `apps/portfolio`), zod 4.4.3,
react-hook-form 7.81.0 (wired in a later plan), vitest for unit tests, Vite 8.1.4.

## Global Constraints

- All colors/UI components live in `packages/ui`; `apps/admin` imports only, never hand-rolls
  markup or raw styles (root `CLAUDE.md` HARD RULE).
- Sanity access happens **only** inside server functions; the token is a server-only env var,
  never `VITE_`-prefixed, never imported by client code.
- Document shapes in `shared/` must exactly match the existing Studio schemas so
  `apps/portfolio` queries keep working unchanged (see
  `docs/superpowers/specs/2026-07-12-admin-studio-invoices-design.md`).
- Languages: `en` (default), `es`, `pt` — matches `apps/portfolio/CLAUDE.md`.
- Biome formatting (single quotes, trailing commas `all`, 2-space indent, 100 col) — do not run
  `pnpm check`/`pnpm format`, just write clean code.
- Do not run `pnpm dev` or `pnpm build` (or start any server) at any point in this plan — the
  user runs those. `pnpm typecheck` and `vitest` (unit tests only) are allowed for verification.
- Never commit unless explicitly instructed.

```typescript
import type { DocumentStatus } from '../../shared/types';
import { draftSanityClient } from './client';

interface RawDoc {
  _id: string;
  [key: string]: unknown;
}

function draftIdOf(id: string): string {
  return id.startsWith('drafts.') ? id : `drafts.${id}`;
}

function publishedIdOf(id: string): string {
  return id.startsWith('drafts.') ? id.slice('drafts.'.length) : id;
}

function resolveStatuses<T extends RawDoc>(docs: T[]): Array<T & { _status: DocumentStatus }> {
  const byPublishedId = new Map<string, { published?: T; draft?: T }>();

  for (const doc of docs) {
    const publishedId = publishedIdOf(doc._id);
    const entry = byPublishedId.get(publishedId) ?? {};
    if (doc._id.startsWith('drafts.')) {
      entry.draft = doc;
    } else {
      entry.published = doc;
    }
    byPublishedId.set(publishedId, entry);
  }

  const result: Array<T & { _status: DocumentStatus }> = [];
  for (const { published, draft } of byPublishedId.values()) {
    if (draft && published) {
      result.push({ ...draft, _status: 'unpublished-changes' });
    } else if (draft) {
      result.push({ ...draft, _status: 'draft' });
    } else if (published) {
      result.push({ ...published, _status: 'published' });
    }
  }
  return result;
}

export async function listDocuments<T extends RawDoc>(
  type: string,
): Promise<Array<T & { _status: DocumentStatus }>> {
  const docs = await draftSanityClient.fetch<T[]>(`*[_type == $type]`, { type });
  return resolveStatuses(docs);
}

export async function getDocument<T extends RawDoc>(
  type: string,
  id: string,
): Promise<(T & { _status: DocumentStatus }) | null> {
  const publishedId = publishedIdOf(id);
  const draftId = draftIdOf(id);
  const docs = await draftSanityClient.fetch<T[]>(
    `*[_type == $type && (_id == $publishedId || _id == $draftId)]`,
    { type, publishedId, draftId },
  );
  const [resolved] = resolveStatuses(docs);
  return resolved ?? null;
}
```

function toDraftId(id: string): string {
  return id.startsWith('drafts.') ? id : `drafts.${id}`;
}

function toPublishedId(id: string): string {
  return id.startsWith('drafts.') ? id.slice('drafts.'.length) : id;
}

export async function publishDocument(id: string): Promise<void> {
  const draftId = toDraftId(id);
  const publishedId = toPublishedId(id);
  const draft = await draftSanityClient.getDocument(draftId);
  if (!draft) {
    throw new Error(`No draft found for ${draftId}`);
  }
  await draftSanityClient
    .transaction()
    .createOrReplace({ ...draft, _id: publishedId })
    .delete(draftId)
    .commit();
}

export async function discardDraft(id: string): Promise<void> {
  const draftId = toDraftId(id);
  await draftSanityClient.transaction().delete(draftId).commit();
}
